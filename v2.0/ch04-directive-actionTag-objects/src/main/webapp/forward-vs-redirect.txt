[ forward ]
고객 (11.jsp) --request--> 상담원1 (12) --request---> 상담원2 (13) --request--> 상담원3 (14)
url: 상담원1 (request 전달 (재사용))


[ redirect ]
고객 (15.jsp) --> 상담원1 (16) --> "전화번호 1234"에 요청하세요! --> 전화 끊음
고객 (15) --> "전화번호 1234" 전화 --> 상담원2 (17)
url: 상담원2 (request 전달 X)


ex) 게시판 글쓰기, 검색, 로그인, 회원가입
forward  - 단순 조회 (request 전달 (재사용)) (검색, 로그인)
redirect - DB 수정 많음 (request 전달 X) (글쓰기, 회원가입)

ex) 만약 글쓰기를 하는데 forward로 작성하게 되면 어떤 문제가 생길까요?
(+ 사용자의 실수로 새로고침을 누를 경우)
=> 글이 여러번 작성이 될 수가 있음
